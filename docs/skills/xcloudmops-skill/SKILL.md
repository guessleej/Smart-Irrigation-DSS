# xCloudMOPs 模組化開發 Skill

## 1. Skill 簡介與目標

本 Skill 旨在為 **xCloudMOPs** 平台的開發提供一個全面、模組化且與最新架構緊密結合的開發指南。它不僅僅是一份靜態文檔，更是一個動態的、可執行的開發框架，旨在指導開發者從前端到後端，從基礎設施到業務邏輯，高效、一致地完成開發、測試與部署任務。

**核心目標**：

*   **架構對齊**：確保所有開發工作都嚴格遵循已定義的微服務、分層架構和技術棧。
*   **開發加速**：提供清晰的模組化開發指南、API 規格和最佳實踐，減少重複工作，加速開發週期。
*   **品質保證**：整合測試策略、除錯指南和部署清單，確保代碼質量和系統穩定性。
*   **體驗一致**：將 UI/UX 設計理念融入前端開發指南，確保最終產品提供專業、直觀、模組化的用戶體驗。

## 2. Skill 結構

本 Skill 採用模組化結構，與 xCloudMOPs 的系統架構一一對應。每個模組都是一個獨立的 Markdown 文件，可以獨立查閱，也可以組合起來形成完整的開發藍圖。

*   **reference/**: 存放核心的架構圖分析和設計理念。
*   **frontend/**: 包含所有前端開發相關的指南，從環境設置到具體的功能實現。
*   **backend/**: 包含後端各微服務的開發指南和 API 規格。
*   **testing/**: 涵蓋單元測試、整合測試和端到端測試的策略與實踐。
*   **deployment/**: 提供關於 Docker、Kubernetes 和 CI/CD 的部署指南。

## 3. 如何使用本 Skill

1.  **從架構開始**：首先閱讀 `reference/architecture_analysis.md` 和 `reference/ui_ux_design.md`，建立對系統全貌和設計理念的宏觀理解。
2.  **分工協作**：前端開發者可以專注於 `frontend/` 目錄下的指南，後端開發者則可以根據自己負責的微服務，查閱 `backend/` 中的相應文件。
3.  **測試驅動**：在開發過程中，隨時參考 `testing/` 目錄下的指南，編寫高質量的測試案例。
4.  **部署交付**：當年功能開發完成後，遵循 `deployment/` 中的流程進行容器化和部署。
