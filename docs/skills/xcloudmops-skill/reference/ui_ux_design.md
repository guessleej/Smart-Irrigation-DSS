# xCloudMOPs UI/UX 設計理念與介面佈局

## 1. 核心設計理念：專業、直觀、模組化

xCloudMOPs 是一個功能強大的 LLMOps 平台，其 UI/UX 設計必須圍繞三大核心理念，以滿足專業開發者和 AI 研究人員的需求，同時降低新手的入門門檻。

*   **專業 (Professional)**：介面設計應體現企業級軟體的嚴謹性與可靠性。採用以 Ant Design 為基礎的設計語言，搭配 Tailwind CSS 進行客製化，打造一個清晰、一致且資訊密度高的介面。Recharts 將用於呈現專業、精確的數據圖表，而 Grafana 的監控面板將被無縫嵌入，提供專業級的系統監控視圖。

*   **直觀 (Intuitive)**：儘管平台功能複雜，但操作流程應盡可能直觀。我們將大量採用視覺化工具，如基於 ReactFlow 的拖拽式畫布，用於 Agent 和工作流的設計。使用者應能透過視覺化操作完成大部分複雜的邏輯編排，而非編寫繁瑣的程式碼。

*   **模組化 (Modular)**：整個平台的功能將被拆分為獨立但互聯的模組。這種模組化設計不僅體現在後端架構上，也將貫穿於前端介面。使用者可以在不同的功能模組之間無縫切換，例如從「知識庫管理」跳轉到「Agent 編輯器」，並將知識庫作為一個模組化的「節點」在 Agent 中使用。

## 2. 整體介面佈局：三欄式佈局

為了在有限的螢幕空間內承載豐富的功能，我們將採用經典且高效的三欄式佈局。

| 左側導航欄 (Navigation) | 中間主工作區 (Workspace) | 右側配置/資訊欄 (Inspector) |
| :--- | :--- | :--- |
| **功能**：提供對平台所有核心模組的快速訪問，如 Dashboard、知識庫、模型、Agent、工作流、監控等。 | **功能**：使用者的主要操作區域。根據所選模組，呈現不同的介面，如 Agent 設計畫布、知識庫文件列表、模型微調儀表板等。 | **功能**：提供對主工作區中選定元素的詳細資訊展示和參數配置。例如，在 Agent 畫布中點擊一個節點，右側欄會顯示該節點的詳細設定選項。 |
| **設計**：可收縮的垂直圖標菜單，懸停時顯示文字標籤，保持工作區最大化。 | **設計**：寬敞、簡潔，是視覺焦點所在。 | **設計**：可根據需要顯示或隱藏，提供上下文相關的操作。 |

這種佈局確保了清晰的功能分區，讓使用者能夠在保持主任務焦點的同時，方便地進行全局導航和細節配置。

## 3. 核心功能 UI/UX 設計概念

基於最新的架構圖，我們為幾個高級功能設計了具體的 UI/UX 概念。

### 3.1 Agent 層：視覺化畫布設計器 (Canvas Designer)

Agent 的創建和編輯將是平台的核心體驗。我們將摒棄傳統的表單配置，採用基於 ReactFlow 的視覺化畫布設計器。

*   **畫布 (Canvas)**：使用者可以在一個無限畫布上，透過拖拽來放置和連接不同的功能「節點」，從而構建一個 Agent 的執行邏輯。
*   **節點 (Nodes)**：每個節點代表一個原子操作，例如「從知識庫檢索」、「呼叫外部 API」、「執行 Python 程式碼」、「生成回覆」等。節點將以卡片形式呈現，清晰地標示其功能和狀態。
*   **組件庫 (Component Library)**：畫布左側或頂部將提供一個「組件庫」，使用者可以從中拖出預設的或自定義的節點到畫布上。這對應了架構中的「組件庫管理」。
*   **狀態管理 (State Management)**：在運行和除錯模式下，節點的邊框會以不同顏色（如綠色表示成功，紅色表示失敗，黃色表示進行中）顯示其執行狀態，數據流也會在連接線上有動態效果，直觀地展示 Agent 的「狀態管理」和「工作流引擎」的執行過程。

### 3.2 GraphRAG 層：知識圖譜探索器 (Graph Explorer)

為了讓使用者能夠理解和利用 GraphRAG 的強大能力，我們需要一個專門的介面來視覺化知識圖譜。

*   **圖譜構建**：提供一個引導式流程，讓使用者上傳非結構化數據，系統在後端進行「實體關係抽取」和「知識圖譜構建」。UI 上會顯示處理進度，並在完成後通知使用者。
*   **視覺化探索**：構建完成的圖譜將在一個交互式的圖形介面中展示。使用者可以縮放、拖動、搜索特定節點（實體），並查看節點之間的關係。點擊一個節點，右側的「配置/資訊欄」會顯示其詳細屬性和關聯資訊。
*   **圖推理搜索**：提供一個自然語言輸入框，使用者可以像對話一樣提出複雜問題（例如「查找所有與 A 公司有關，並且在過去一年內發生過投資事件的實體」）。系統將在後端執行「圖推理搜索」和「多跳推理」，並在圖譜上高亮顯示相關的路徑和節點，將推理過程視覺化。

### 3.3 工作流編排層：企業級流程編排器 (Orchestrator)

此功能面向更複雜的、跨越多個服務的自動化任務，是 Agent 設計器的擴展和升級版。

*   **統一介面**：與 Agent 設計器共享相似的視覺化畫布 UI，但提供更強大的控制邏輯節點。
*   **高級節點**：除了 Agent 的基礎節點外，還提供「條件分支 (If/Else)」、「並行處理 (Parallel)」、「錯誤處理 (Try/Catch)」、「定時觸發」等高級流程控制節點，對應架構中的「任務調度引擎」和「條件分支邏輯」。
*   **流程監控與恢復**：在 Dashboard 或專門的監控頁面，使用者可以查看每個工作流的執行歷史、狀態和日誌。對於失敗的任務，UI 上應提供「重試」或「從失敗步驟恢復」的選項，對應「錯誤處理恢復」機制。

## 4. 技術棧整合

*   **前端框架**：整個應用將基於 UmiJS 4.0 + React 18 + TypeScript 構建，確保開發效率和代碼質量。
*   **UI 組件**：Ant Design 5.x 作為基礎組件庫，Radix UI 用於構建需要更深度客製化的無頭 (headless) 組件，Tailwind CSS 負責樣式微調和響應式設計。
*   **狀態管理**：Zustand 用於管理全局客戶端狀態（如主題、用戶資訊），而 TanStack Query (React Query) 將作為核心的服務端狀態管理器，負責處理所有與後端 API 的數據交互、緩存和同步，完美匹配 Redis 的緩存策略。
*   **視覺化**：ReactFlow 用於 Agent 和工作流畫布，Recharts 用於數據圖表，PDF.js 或類似庫整合用於文件預覽。

透過這套精心設計的 UI/UX 理念和介面佈局，xCloudMOPs 將不僅僅是一個功能集合，而是一個體驗流暢、功能強大且易於擴展的專業級 AI 開發與運維平台。
